# Osteoporosis ML Predictor Website PRD

## 1. 项目背景

投稿期刊要求预测模型具备可访问的网站或在线工具，用于展示模型用途、输入变量、验证结果和临床应用边界。本网站服务于骨质疏松机器学习预测模型的论文投稿与读者复核，重点体现模型的可解释性、外部验证结果和筛查决策价值。

当前论文模型为基于 NHANES 训练、KNHANES 外部验证的三分类骨密度状态预测模型，目标类别为 Normal、Osteopenia 和 Osteoporosis。最终稿件模型为堆叠模型，基础学习器包括 ExtraTrees、GradientBoosting、Bagging、LightGBM 和 XGBoost，元学习器为 Logistic Regression。

## 2. 产品目标

1. 让审稿人和读者快速理解模型的研究背景、变量输入、验证性能和临床使用边界。
2. 提供一个美观、可信、期刊友好的在线展示页面，视觉风格参考极简科技/医疗机构网站。
3. 提供独立风险计算页，用于展示预测模型的交互形式。
4. 明确区分“网页演示模式”和“正式模型服务模式”，避免临床误用。

## 3. 目标用户

- 期刊编辑和审稿人：验证模型是否具备可展示、可访问、可解释的在线界面。
- 研究读者：理解模型输入变量、预测任务和验证结果。
- 临床研究者：评估模型作为 DXA 筛查前置工具的潜在价值。
- 后续开发者：根据开发规范接入真实模型服务。

## 4. 页面范围

### 4.1 首页展示

- 顶部导航：Model、Evidence、Calculator、Methods。
- Hero 区域：说明模型定位为 osteoporosis screening risk stratifier。
- 核心指标：NHANES internal macro AUC、KNHANES external macro AUC、DXA referral reduction、post-calibration ECE。
- 医疗/研究图像带：营造正式科研工具气质。

### 4.2 Evidence 区域

- 展示模型训练来源、外部验证来源、三分类任务、与 OST 的比较结论。
- 需强调：模型整体判别能力和分类准确性优于 OST，但 OST 在骨质疏松高敏感筛查方面仍有价值。

### 4.3 Calculator 页面

- 提供年龄、性别、种族/族群、身高、体重、腰围、血压、ALP、钙、肌酐、活动水平等输入。
- 自动计算 BMI 和 OST。
- 输出 Normal、Osteopenia、Osteoporosis 三类风险比例。
- 给出 DXA referral 建议等级。
- 独立计算页优先调用本地模型 API。API 可用时实际加载保存的堆叠模型；API 不可用时回退到 frontend demonstration score。

### 4.4 Methods 区域

- 只说明工具用途、研究边界和非诊断声明。
- 不在公开首页放论文性能表、DXA 换算表或部署说明；这些内容保留在论文、补充材料和审稿回复中。

### 4.5 Footer

- 提供研究用途声明、模型版本、联系方式占位。

## 5. 功能需求

| 模块 | 需求 |
|---|---|
| 导航 | 点击导航平滑滚动到对应区块 |
| 指标展示 | 以四列统计数字呈现核心验证结果 |
| 计算器 | 支持输入校验、BMI 自动计算、OST 自动计算、三分类风险输出 |
| 结果解释 | 显示主预测类别、概率条、DXA referral 建议 |
| 免责声明 | 明确非诊断工具，不替代 DXA 或医生判断 |
| 响应式 | 桌面、平板、手机均需可读可用 |

## 6. 数据与模型边界

当前网站包含静态前端和本地模型 API 两部分。静态前端不能直接加载本地 `.pkl` 模型；独立计算页通过 `model_service.py` 调用后端 API 实际加载保存的堆叠模型。由于论文正式堆叠模型依赖与训练阶段一致的缺失值填补、编码和标准化流程，正式生产部署时应将完整 preprocessing pipeline 序列化并锁定版本。

正式部署版本应满足：

1. 后端固定加载 `web/model/model.pkl`。
2. 后端同时加载训练时一致的特征顺序、编码器、填补器和标准化参数。
3. 前端只提交原始临床变量，后端完成预处理和预测。
4. 后端返回三类概率、预测类别、模型版本和预处理版本。
5. 所有预测结果记录匿名日志，用于审计而非临床诊断。

## 7. 验收标准

1. 打开网站首屏可直接看到模型名称、研究定位、核心验证指标和医学视觉素材。
2. 计算器输入完整后可得到三分类风险、BMI、OST 和推荐说明。
3. 页面内所有“10,000 人”相关表述均使用“换算”或“scaled”，不得写成新增真实模拟队列。
4. 页面中必须出现研究用途和非临床诊断声明。
5. 移动端宽度 390px 下导航、表单和结果条不得重叠。
6. 种族/族群变量只用于研究模型校准和跨人群适用性展示，不得写成单独诊断依据。
